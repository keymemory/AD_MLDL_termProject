> 환경: LLaVA-1.5-7B · greedy(temp=0) · fp16. 결과 원자료: `results_phase3_merge.tsv`,

## 0. Phase 3는 왜 하는가 (배경)

Phase 2 묶음 A(전 구간 성능 곡선)에서 **두 가지 숙제**가 남았다.

1. **약점 발견**: TextVQA(OCR)는 **토큰을 많이 병합할수록 성능이 떨어졌다.** 글자 토큰이 배경과
   함께 평균으로 뭉개지기 때문. → 이 약점을 **해결하는 방법**이 필요하다. (실험 A)
2. **정당성 구멍**: 기존 실험은 "M1 = M2의 2배(예 128)"를 **임의로 고정**했다. "왜 하필 128인가?"에
   답하지 못하면 정당성이 약하다. → 고정값이 adaptive의 특수한 경우임을 보여야 한다. (실험 B)

Phase 3는 이 둘을 각각 **실험 A(task-aware 병합)**와 **실험 B(τ* 역산)**로 해결해,
연구의 노벨티를 "발견"에서 "해법"으로, 그리고 정당성을 완결한다.

| 실험 | 카테고리 | 숙제 |
|---|---|---|
| A | **Stage 2 — 병합(merging) 방법** | TextVQA 병합 손상 해결 |
| B | 계산(추론 없음) | 고정 M1의 정당성 |

# 실험 A — Task-Aware (Merge-Distortion) 병합

## A-1. 왜 이 실험을 하는가
묶음 A에서 **TextVQA는 병합이 강할수록(τ↑, M1:M2↑) 성능이 하락**하는 볼록 곡선이 나왔다.
예: M2=32에서 τ=0.5(53.63) → τ=0.8(52.39)로 떨어짐. 원인은 **평균 병합이 글자를 뭉개는 것**.
GQA·POPE(의미 태스크)는 병합이 이득이었는데 TextVQA(OCR)만 손해였다. 이 **OCR 약점을 해결**하지
못하면 "task별 최적이 다르다"는 발견이 문제 지적에 그친다. 그래서 해법을 만든다.

## A-2. 목적
병합 시 정보 손실이 큰 토큰(글자로 추정)을 **자동으로 보존**해서,
(1) TextVQA(OCR) 성능을 회복하고, (2) POPE·GQA(의미 태스크)는 해치지 않는지 검증.

## A-3. 방법과 직관 (핵심)

> **아이디어: "이 토큰은 평균에 넣으면 손실이 크겠다" 싶은 토큰을 자동으로 찾아서,
> 그건 병합 안 하고 원본 그대로 남긴다. 나머지 평범한 토큰만 평균낸다.**

### 왜 필요한가 (일반 병합의 문제)
Stage 2는 비슷한 토큰들을 K-Means로 묶어 **각 그룹을 평균 하나로 대체**한다. 그런데 평균은
**소수의 특이한 토큰을 뭉갠다.** OCR에서 글자 토큰은 주변 배경과 성격이 완전히 다른데,
배경 토큰들과 한 그룹으로 평균되면 **글자 정보가 흐려져 사라진다.**

### 어떻게 "손실 큰 토큰"을 찾나 — 병합 왜곡(distortion)
각 토큰이 **자기 그룹 대표(centroid)와 방향이 얼마나 다른지**를 잰다. 많이 다르면 = 대표로
대체했을 때 손실이 큼 = 보존해야 함.

```
Distortion(i) = 1 − cos(x_i, c_{cluster(i)})           # 토큰 x_i와 그 클러스터 centroid c의 방향 차이
preserve = { i : Distortion(i) ≥ μ_d + k_d·σ_d }        # 왜곡이 통계적 이상치인 토큰 → 원본 보존
```
- **평범한 토큰**(배경): centroid와 비슷 → 왜곡 작음 → 그냥 평균 병합
- **특이한 토큰**(글자 획): centroid와 확 다름 → 왜곡 큼 → **이상치로 잡혀 원본 보존**
- `k_d`: 이상치 임계(높을수록 적게 보존). 스윕 대상.

### 최종 토큰 수 불변 (공정 비교)
```
preserve p개(원본 그대로) + 나머지를 (M2 − p)개로 병합  =  M2개
```
K-Means를 M2가 아니라 (M2−p)개로 돌려서, 보존한 p개와 합치면 정확히 M2가 된다.
`p ≤ M2·0.5` 상한(넘으면 상위 왜곡만 보존).

### 왜 "task-aware"인가
명시적으로 "이건 글자다"라고 안 알려줘도, OCR에선 글자 토큰이 자연히 왜곡이 커서
**자동 보존**된다. 즉 병합이 태스크(OCR)의 특성에 **스스로 맞춰진다(aware).**

## A-4. 데이터
- **TextVQA(핵심)**: OCR 태스크. 회복을 봐야 할 대상.
- **POPE·GQA(대조군)**: 의미 태스크. task-aware가 이들을 **해치지 않는지** 확인용.
- **baseline**: 묶음 A energy τ=0.8 weighted (일반 병합) — 같은 조건에서 병합 방법만 바꿔 비교.

## A-5. 실험 조건
merge_method=taskaware, k_d ∈ {0.5, 1.0, 1.5, 2.0}, M2 ∈ {32, 64, 128}, selection=energy τ=0.8.
(τ=0.8은 병합이 많아 OCR 손상이 큰 조건 → 개선 효과가 잘 드러남.) 총 36 job.

## A-6. 결과 (best Δ = taskaware 최고 − weighted baseline)
| 데이터셋 | M2=32 | M2=64 | M2=128 | 판정 |
|---|---|---|---|---|
| **TextVQA**(핵심 OCR) | **+0.15** | **+0.25** | **+0.38** | ✅ 모든 M2 회복 |
| POPE(대조) | −0.007 | +0.003 | +0.003 | ✅ 중립 |
| GQA(대조) | −0.55 | +0.11 | +0.29 | 🟡 M2=32만 근소 하락 |

- TextVQA 예시(M2=128): weighted 55.51 → taskaware(k_d=1.0) **55.89**.
- preserve된 평균 토큰 수: M2=32에서 8~16개, M2=64에서 27~32개(M2의 25~50%).

## A-7. 효과·장단점
- **효과(장점)**: **TextVQA(OCR)를 모든 M2에서 회복**(+0.15~0.38). 병합에서 글자 토큰을 자동 보존한
  결과. POPE 중립·GQA 대체로 중립 → **약점을 고치면서 강점은 안 해침**. 노벨티가 "문제 발견 → 해법"으로 완결.
- **장점(설계)**: OCR 모델·라벨 없이 **feature 왜곡만으로 글자를 자동 감지**. M2 불변이라 공정 비교·zero overhead.
- **한계(단점)**:
  1. 개선폭이 작다(+0.15~0.38) — 극적 회복은 아님.
  2. GQA M2=32에서 −0.55 — 강압축에선 보존이 오히려 다른 중요 정보를 밀어냄.
  3. 최적 k_d가 M2·데이터셋마다 달라(예: TextVQA M2=128은 k_d=1.0) 튜닝 필요.
- **향후**: 텍스트 영역 마스킹과 결합, k_d 자동 결정, 태스크 유형별 병합 라우팅.

# 실험 B — τ* 역산 (왜 고정값 M1=128인가)

## B-1. 왜 이 실험을 하는가
기존 실험은 "M1 = M2의 2배(예 128)"를 **임의로 고정**했다. "왜 하필 128인가?"에 답하지 못하면,
"그 값이 우연히 좋았던 것 아니냐"는 반박에 취약하다. 고정값이 사실 **adaptive의 특수한 한 점**임을
보이면, adaptive가 고정을 **일반화**한다는 정당성이 완성된다.

## B-2. 목적
각 데이터셋에서 **평균 M1이 128(=고정 topk M1)이 되는 energy τ***를 역산 →
"고정 = adaptive의 τ=τ* 특수해"임을 수치로 증명.

## B-3. 방법과 직관
τ를 올리면 M1이 자동으로 커진다(실험 5에서 이미 τ별 평균 M1을 다 측정). 그 측정점들을 보간해
**"평균 M1이 정확히 128이 되는 τ"**를 찾으면 그게 τ*다. **추가 추론이 전혀 필요 없다**(기존 분포 재활용).

```
τ* = (τ, 평균M1) 측정점들을 선형 보간해 평균M1 = 2×M2(=고정 M1) 되는 τ
```

## B-4. 데이터
묶음 A의 τ별 평균 M1 (POPE/GQA/TextVQA × M2=32/64/128). 추가 추론 0.

## B-5. 실험 조건
각 (데이터셋, M2)에서 평균 M1 = 2×M2 되는 τ* 선형 보간.

## B-6. 결과
| 데이터셋 | τ* (M1=128, M2=64) | M2별 τ* (M1=2×M2) |
|---|---|---|
| POPE | 0.646 | 0.52 / 0.65 / 0.79 |
| GQA | 0.637 | 0.52 / 0.64 / 0.78 |
| TextVQA | 0.692 | 0.55 / 0.69 / 0.83 |

- 검증: POPE M2=64에서 τ*=0.646 지점 성능 ≈ 0.815 ≈ 고정 topk 성능(0.8227)로 근접(실험 2와 일관).

## B-7. 효과·장단점
- **효과(장점)**: 고정 M1=128이 **데이터셋마다 다른 τ*(0.637~0.692), M2마다도 다른 τ*(0.52~0.83)**에
  해당 → **고정값은 임의적이며, adaptive는 τ 하나로 그 스펙트럼 전체를 파라미터화하고 고정을 특수해로
  포함**. "왜 128인가"에 "각 조건의 τ*일 뿐"이라 답.
- **장점**: **추론 비용 0**(계산만)으로 정당성 논증 완결.
- **한계(단점)**: 계산 기반이라 τ* 근처 실제 성능 검증을 1점만 함(더 촘촘히 하면 완결성↑).

## Phase 3 종합

| 실험 | 숙제 | 결과 | 기여 |
|---|---|---|---|
| A (task-aware 병합) | TextVQA 병합 손상 | **OCR 전 M2 회복**(+0.15~0.38), 의미 태스크 중립 | ★ 발견→**해법** |
| B (τ* 역산) | 고정 M1 정당성 | 고정 = adaptive의 τ* **특수해** | 정당성 완결 |

- **실험 A**는 "task별 최적이 다르다"(Phase2 발견)를 **"그럼 어떻게 고칠까"(해법)**로 확장 — task-aware
  병합으로 OCR 손상을 자동 회복.
- **실험 B**는 "왜 고정 M1인가"에 답해, adaptive가 기존 고정 방식을 **일반화**함을 증명.
- 두 실험으로 Phase 2의 두 숙제(약점·정당성)를 모두 해소하고, 논문의 노벨티와 정당성을 함께 완결한다.
